= Sécurité
:page-languages: [en, ja, ko, zh]
:revdate: 2025-12-31
:page-revdate: {revdate}

Cette section décrit la méthodologie et les moyens de sécuriser un cluster K3s. Il est divisé en 2 sections. Ces guides supposent que k3s fonctionne avec etcd intégré.

Tout d'abord, le guide de renforcement de la sécurité fournit une liste de meilleures pratiques de sécurité pour sécuriser un cluster K3s.

* xref:security/hardening-guide.adoc[Guide de renforcement de la sécurité]

Deuxièmement, il y a l'auto-évaluation pour valider un cluster renforcé. Nous avons actuellement deux évaluations différentes disponibles :

* xref:security/self-assessment-1.9.adoc[CIS 1.9 Benchmark Self-Assessment Guide], pour la version K3s v1.27-v1.29
* xref:security/self-assessment-1.10.adoc[CIS 1.10 Benchmark Self-Assessment Guide], pour la version K3s v1.28-v1.31
* xref:security/self-assessment-1.11.adoc[CIS 1.11 Benchmark Self-Assessment Guide], pour la version K3s v1.29-v1.34
