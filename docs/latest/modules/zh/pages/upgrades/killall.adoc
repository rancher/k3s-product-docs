= 停止 K3s
:page-languages: [en, ja, ko, zh]
:revdate: 2024-09-26
:page-revdate: {revdate}

为了在升级期间实现高可用性，当 K3s 服务停止时，K3s 容器将继续运行。

== K3s 服务

安装脚本支持 systemd 和 OpenRC 的 K3s 停止和重启。

[tabs]
======
systemd::
+
--
停止服务器：

[,sh]
----
sudo systemctl stop k3s
----

重启服务器：

[,sh]
----
sudo systemctl start k3s
----

停止代理：

[,sh]
----
sudo systemctl stop k3s-agent
----

重启代理：

[,sh]
----
sudo systemctl start k3s-agent
----
--

OpenRC::
+
--
停止服务器：

[,sh]
----
sudo rc-service k3s stop
----

重启服务器：

[,sh]
----
sudo rc-service k3s restart
----

停止代理：

[,sh]
----
sudo rc-service k3s-agent stop
----

重启代理：

[,sh]
----
sudo rc-service k3s-agent restart
----
--
======

== Killall 脚本

要停止所有 K3s 容器并重置 containerd 状态，可以使用 `k3s-killall.sh` 脚本。

killall 脚本清理容器、K3s 目录和网络组件，同时移除所有相关规则的 iptables 链。集群数据不会被删除。

要从服务器节点运行 killall 脚本，请运行：

[,bash]
----
/usr/local/bin/k3s-killall.sh
----
